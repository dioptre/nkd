Known Problems.

 

Some general known problems users have asked.

 

Problem: Can't relay from client computers.

Solution: This is because relay is enabled for client computer IP.

  • Allow relay for IP or IP range (in manager).
  • Setup SMTP authentication in SMTP Client. User names and passwords are same as in POP3 account.

 

 

Problem: Can't relay in server computer using localhost.

Solution: This is because relay is enabled for localhost.

  • Allow relay for IP 127.0.0.1 (localhost).
  • Setup SMTP authentication in SMTP Client. User names and passwords are same as in POP3 account.

 

 

Problem: Can't access SMTP/POP3 from LAN or WAN.

Solution: Check if in server security SMTP/POP3 access is enabled for your connecting IP.

 

 

Problem: Can't receive messages, but can telnet to public IP and MAIL FORM:<xxx>, RCPT TO:<your_adress@domain.xx> shows "OK".

Solution: Check that you have MX record associated with your domain.

 

 

Problem: Can't receive messages and get 550 reply, but user and domain exists.

Solution: After register.bat you must get mailServiceError.log with line "System:Only one usage of each socket address (protocol/network address/port) is normally permitted". Check that you are not running another SMTP Server software like IIS SMTP.
